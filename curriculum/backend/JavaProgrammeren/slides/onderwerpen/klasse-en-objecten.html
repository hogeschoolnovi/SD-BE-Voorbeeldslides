Het probleem

Stel je wilt van 10 personen de voornaam, achternaam en leeftijd bijhouden. Hoe ga je dat doen?

Dertig variabele met namen als name1, lastname1, age1 is erg onhandig!

Java heeft klassen (class) om data te groeperen!

Theorie - Algemeen

  • Java is object-georiënteerd.
  • Een object gebruikt een klasse als blauwdruk
  • Van één klasse kun je dus meerdere objecten maken.

Theorie - Klasse (1)

  • Zo'n klasse is dus een blauwdruk van een object
  • Het definieert welke data een object kan bevatten en welk gedrag een object kan doen
  • De data die het kan bevatten noemen we instantie variabelen (instance variables)
  • Het gedrag schrijven we in methodes

Theorie - Klasse (2)

  • Klassenaam is dezelfde als bestandsnaam
  • Klassenaam begint met een hoofdletter
  • Instance variable-namen beginnen met een kleine letter
  • Methodes beginnen met een kleine letter

Klasse - code (1)

  • Maak een klasse aan. Noem deze Person

        public class Person {
            public String firstName;
            public String lastName;
            public int age;
        }

Normaliter zijn instance-variables private, maar voor het voorbeeld nu even niet.

Klasse - Object maken!

Dit doen we in de main-methode.


        public static void main(String[] args) {
            Person aPerson = new Person();
            aPerson.firstName = "Karel";
            aPerson.lastName = "de Grote";
            aPerson.age = 538;

            System.out.println("Persoon: "+ aPerson.firstName + " "
                + aPerson.lastName + " is " + aPerson.age);
        }

De constructor - Theorie

  • Om van een klasse een object te maken gebruik je altijd de constuctor
  • De constructor wordt aangeroepen met het new keyword
  • Wanneer je geen constructor aan een klasse toevoegt, voegt Java automatisch een lege constructor toe.

De constructor - Theorie (2)

  • Een constructor lijkt op een methode, maar is het niet.
  • Een constructor heeft bijvoorbeeld geen return-type
  • Dus ook geen void
  • De constructor heeft dezelfde naam als de klasse!

De constructor - Lege constructor


        public class Person {
            public String firstName;
            public String lastName;
            public int age;

            public Person() {
            }
        }

Constructor - Alle parameters


        public class Person {
            public String firstName;
            public String lastName;
            public int age;

            public Person(String firstName, String lastName, int age) {
                this.firstName = firstName;
                this.lastName = lastName;
                this.age = age;
            }
        }

Object aanmaken - Constructor met parameters


        Person anotherPerson = new Person("Filips", "De Tweede", 496);

        System.out.println("Persoon: "+ anotherPerson.firstName + " "
            + anotherPerson.lastName + " is " + anotherPerson.age);

Constructors - dit moet je weten

  • Je maakt van een klasse een object met een constructor
  • Wanneer je geen constructor toevoegt, voegt Java een lege constructor toe.
  • Een klasse kan meerdere constructor hebben, zolang de datatypes van de parameters maar verschillen.
  • Een constructor is bijna altijd public
  • Constructor-naam is dezelfde als klasse-naam
  • Geen return-type en void.

Constructors - Volledige Main


        public static void main(String[] args) {
            Person aPerson = new Person();
            aPerson.firstName = "Karel";
            aPerson.lastName = "de Grote";
            aPerson.age = 538;

            System.out.println("Persoon: "+ aPerson.firstName + " "
                + aPerson.lastName + " is " + aPerson.age);

            Person anotherPerson = new Person("Filips", "De Tweede", 496);
            System.out.println("Persoon: "+ anotherPerson.firstName + " "
                + anotherPerson.lastName + " is " + anotherPerson.age);
            }

Methodes

Getters & Setters

  • In de regel zijn instance variables altijd private
  • Private variables en methodes kunnen alleen binnen de klasse zelf aangeroepen worden
  • I.a.w: Een main-methode kan dan dus niets doen.
  • Daarom maken we public methodes, die instance variables kunnen uitlezen en aanpassen.
  • Uitlezen is: get
  • Aanpassen is: set

Getters & Setters (2)

  • Een get methode heet: getNaamVanVariable()
  • Een set methode heet: setNaamVanVariable()

Getters & Setters (3)


            public class Person {
                private String firstName;
                private String lastName;
                private int age;

                public String getFirstName() {
                    return firstName;
                }
                public void setFirstName(String firstName) {
                    this.firstName = firstName;
                }
            }

Voeg zelf de getter en setter van lastName en Age toe.

Getters & Setters - Gebruiken


        public class MainMethods {
            public static void main(String[] args) {
                Person p = new Person();

                p.setFirstName("Willem");
                p.setLastName("van Oranje");
                p.setAge(512);

                System.out.println("Persoon: "+ p.getFirstName() + " "
                    + p.getLastName() + " is " + p.getAge());
                }
        }
