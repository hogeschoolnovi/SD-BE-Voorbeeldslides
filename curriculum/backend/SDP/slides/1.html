Waarom git?

  • Voorkomt code-overschrijvingen
  • Het onthoudt alle aanpassingen die gedaan zijn
  • Meerdere mensen kunnen tegelijk aan hetzelfde project (en bestand) werken
  • Repositories staan op meerdere systemen.
  • Versiebeheer, dus makkelijke code-reviewen
  • Geeft de mogelijkheid om te werken met pull-requests.

Clonen

Fork

Het downloaden/clonen van een remote repository naar een andere remote repository, zodat jij kunt comitten en pushen.

Fork

  • Fork is geen git-commando
  • Wordt aangeboden door schillen (zoals GitHub, GitLab, BitBucket)
  • De geforkte repo moet je uiteindelijk weer clonen naar je lokale repo

Forken in GitHub

  • https://github.com/hogeschoolnovi/git-example

Forken - zelf

  • Fork de gegeven repo en clone deze vervolgens naar je lokale machine.

Comitten en pushen

  • We gebruiken het commit-commando om dingen op te slaan in onze lokale repo.
    • Dit kan alleen nadat bestanden ge-add zijn!
  • We gebruiken de push-commando om jouw code te plaatsen op de remote repo.

C & P - CLI

  • git add index.html
  • git commit "Added index.html"
  • git push origin master

Samenwerken

Samenwerken - regels

  • Niet pushen op de master-branch
  • Master-branch bevat productie-code
  • Branchen altijd vanaf development-branch
  • Werken in feature-branches.
  • Er wordt alleen gemergd na een code review op een pull-request
  • Maximaal 1 persoon pusht op een branch

Samenwerken - Jargon

  • Branch: Aftakking van punt in de code waar 1 persoon op werkt
  • Master-branch: Code die in productie draait.
  • Development-branch: Gebrancht vanaf Master. Wordt de nieuwe master-branch
  • Feature-branch: Aftakking van Development. Bevat specifieke implementatie van een feature.
  • Mergen: Het samenvoegen van twee branches.
  • Code review: Het controleren van (de werking en) de kwaliteit van andermans code.
  • Pull request: Verzoek tot mergen van de branch en verzoek tot code-review
